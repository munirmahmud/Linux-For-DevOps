# Chapter 4 - Lesson 10: Network File Configuration

**Chapter 4 | Lesson 10 of 11**

Deep Dive about /etc/hosts, /etc/resolv.conf, /etc/network/interfaces


## 🎯 এই Lesson-এ কী শিখবো?

এই lesson-এ আমরা Linux-এর তিনটি অত্যন্ত গুরুত্বপূর্ণ networking configuration file সম্পর্কে বিস্তারিত জানবো:

1. **`/etc/hosts`** - Local DNS table
2. **`/etc/resolv.conf`** - DNS resolver configuration
3. **`/etc/network/interfaces`** - Network interface configuration (Debian/Ubuntu)

এই তিনটি file একজন DevOps Engineer-এর কাছে **daily কাজের** অংশ। Server setup, troubleshooting, container networking - সব জায়গায় এগুলো কাজে আসে।


## PART 1: `/etc/hosts`

### এটা কী?

ধরুন আপনি কাউকে চিনেন তার নাম দিয়ে যেমন "রাতুল" বললেই বুঝতে পারেন যে সে কে। কিন্তু officially তার একটা ID number আছে।

`/etc/hosts` হলো ঠিক সেরকম একটা `local phonebook` যেখানে আপনি নিজে লিখে রাখতে পারেন:

> "এই IP address-টার নাম হবে `myserver`"

যখনই আপনি `myserver` লিখবেন, Linux DNS-এ না গিয়ে সরাসরি এই file থেকে IP বের করে নিবে।

### File দেখা

```bash
cat /etc/hosts
```

**Expected Output:**
```
127.0.0.1       localhost
127.0.1.1       my-ubuntu-pc
::1             localhost ip6-localhost ip6-loopback
```

### প্রতিটি অংশের মানে

| Column | মানে |
|--------|------|
| `127.0.0.1` | IP address |
| `localhost` | Hostname (নাম) |
| `my-ubuntu-pc` | Machine-এর নিজের নাম |
| `::1` | IPv6 version of localhost |

### নিজে Entry যোগ করা

```bash
sudo nano /etc/hosts
```

নিচে এই line যোগ করুন:

```
192.168.1.50    webserver
192.168.1.51    dbserver
192.168.1.52    devbox
```

এখন test করুন:

```bash
ping webserver
```

**Output:**
```
PING webserver (192.168.1.50) 56(84) bytes of data.
```

এখন DNS server-এ না গিয়েই নাম resolve হয়ে গেল!

### Resolution Order কীভাবে কাজ করে?

Linux যখন কোনো hostname resolve করতে চায়, সে এই order-এ দেখে:

```
1. /etc/hosts (সবার আগে)
2. DNS server (/etc/resolv.conf থেকে)
3. অন্যান্য sources
```

এই order control করে `/etc/nsswitch.conf` file:

```bash
cat /etc/nsswitch.conf | grep hosts
```

**Output:**
```
hosts:          files dns
```

এখানে `files` মানে `/etc/hosts`, `dns` মানে DNS server। `files` আগে আছে, তাই `/etc/hosts` সবার আগে check হয়।

### DevOps-এ `/etc/hosts` কোথায় কাজে লাগে?

**Scenario 1: Development Environment**
```
# Local development এ production domain test করা
127.0.0.1    myapp.production.com
```
এখন browser-এ `myapp.production.com` লিখলে local machine-এ যাবে।

**Scenario 2: Internal Server Names**
```
10.0.0.10    jenkins
10.0.0.11    nexus
10.0.0.12    sonarqube
```
এখন `ssh jenkins` লিখলেই কাজ হবে, IP মনে রাখতে হবে না।

**Scenario 3: DNS Override (Blocking)**
```
0.0.0.0    ads.google.com
0.0.0.0    tracking.facebook.com
```
এই IP-তে কিছু নেই, তাই request fail হবে - ad blocking এভাবেই কাজ করে!

**Scenario 4: Kubernetes / Docker**

Container orchestration system গুলো internally `/etc/hosts` modify করে service discovery করে।

```bash
# একটা running container-এর /etc/hosts দেখা
docker exec <container_id> cat /etc/hosts
```


## PART 2: `/etc/resolv.conf`

### এটা কী?

`/etc/hosts`-এ যদি কোনো entry না থাকে, তখন Linux একটা **DNS Server**-কে জিজ্ঞেস করে "এই নামের IP কী?"

`/etc/resolv.conf` বলে দেয় "কোন DNS Server-কে জিজ্ঞেস করতে হবে"।

> আপনি যখন কারো নম্বর না জানেন, তখন directory assistance বা Google-এ খোঁজো। `resolv.conf` হলো সেই "কোথায় খুঁজবো" এর নির্দেশনা।

### File দেখা

```bash
cat /etc/resolv.conf
```

**Output (typical):**
```
# Generated by NetworkManager
nameserver 8.8.8.8
nameserver 8.8.4.4
search example.com office.local
options timeout:2 attempts:3
```

### প্রতিটি Directive বিস্তারিত


#### 1. `nameserver` - DNS Server Address

```
nameserver 8.8.8.8
nameserver 8.8.4.4
```

- **`nameserver`** - এই IP-তে DNS query পাঠাবে
- **`8.8.8.8`** - Google-এর Public DNS
- **`8.8.4.4`** - Google-এর Backup DNS
- সর্বোচ্চ **3টি** nameserver দেওয়া যায়
- প্রথমটা fail করলে দ্বিতীয়টা try করবে

**Common DNS Servers:**

| Provider | Primary | Secondary |
|----------|---------|-----------|
| Google | 8.8.8.8 | 8.8.4.4 |
| Cloudflare | 1.1.1.1 | 1.0.0.1 |
| OpenDNS | 208.67.222.222 | 208.67.220.220 |
| Local Router | 192.168.1.1 | - |


#### 2. `search` - Search Domain

```
search example.com office.local
```

এটা একটু মজার concept। ধরুন আপনার company-র domain হলো `devopsquad.com`।

আপনি যদি শুধু `ping webserver` লিখেন, Linux automatically try করবে:
- `webserver.devopsquad.com`
- `webserver.office.local`

তাই short name লিখলেই full domain resolve হয়।

**Example:**
```bash
# search domain: devopsquad.com সেট থাকলে
ping jenkins
# আসলে এটা resolve করে: jenkins.devopsquad.com
```


#### 3. `domain` - Local Domain Name

```
domain devopsquad.com
```

`search`-এর মতোই, কিন্তু শুধু একটা domain। `search` এবং `domain` একসাথে থাকলে শেষেরটা কার্যকর হয়।


#### 4. `options` - Advanced Options

```
options timeout:2 attempts:3 rotate
```

| Option | মানে |
|--------|------|
| `timeout:2` | DNS server-কে ২ সেকেন্ড সময় দাও |
| `attempts:3` | ৩ বার try করো, তারপর fail |
| `rotate` | Multiple nameserver-এর মধ্যে load balance করো |
| `ndots:5` | কতটা dot থাকলে absolute name ধরবে |


### Modern Linux-এ একটা বড় বিষয়!

Modern Ubuntu/Debian-এ `resolv.conf` automatically **overwrite** হয়। কারণ:

- **NetworkManager** - GUI network management করে
- **systemd-resolved** - DNS caching service চালায়

```bash
# দেখুন এটা আসলে কোথায় point করছে
ls -la /etc/resolv.conf
```

**Output:**
```
lrwxrwxrwx 1 root root 39 /etc/resolv.conf -> ../run/systemd/resolve/stub-resolv.conf
```

এটা একটা **symbolic link**! Actually `systemd-resolved` manage করছে।

```bash
# systemd-resolved এর status দেখুন
systemctl status systemd-resolved

# DNS settings বিস্তারিত দেখুন
resolvectl status
```

**Output:**
```
Global
       LLMNR setting: no
MulticastDNS setting: no
  DNSOverTLS setting: no
      DNSSEC setting: no
    DNSSEC supported: no
          DNS Servers: 8.8.8.8 8.8.4.4
```

### Manually DNS Change করা (Permanently)

```bash
# systemd-resolved configure করুন
sudo nano /etc/systemd/resolved.conf
```

```ini
[Resolve]
DNS=1.1.1.1 8.8.8.8
FallbackDNS=8.8.4.4
Domains=devopsquad.com
```

```bash
# Restart করুন
sudo systemctl restart systemd-resolved
```

### DNS Test করা

```bash
# Current DNS দিয়ে resolve test
nslookup google.com

# Specific DNS server দিয়ে test
nslookup google.com 1.1.1.1

# dig দিয়ে বিস্তারিত দেখুন
dig google.com @8.8.8.8
```


## PART 3: `/etc/network/interfaces`

### এটা কী?

এটা হলো **Debian/Ubuntu** (non-NetworkManager) system-এ network interface configure করার file।

> আপনার বাড়ির internet connection-এর settings যেমন IP কী হবে, Gateway কী, কোন interface use করবে - সব এখানে লেখা থাকে।

> ⚠️ **Note:** Modern Ubuntu Desktop-এ NetworkManager এবং Netplan বেশি ব্যবহার হয়। কিন্তু Server এবং container environment-এ এই file এখনো অনেক জায়গায় দেখা যায়। তাই জানাটা জরুরি।

### File দেখুন

```bash
cat /etc/network/interfaces
```

**Basic Output:**
```
# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

source /etc/network/interfaces.d/*

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet dhcp
```

### প্রতিটি Line বুঝি


#### `auto` - Boot-এ Automatically Start

```
auto eth0
```
মানে: system start হলে `eth0` interface automatically উঠবে। না থাকলে manually `ifup eth0` করতে হবে।


#### `iface` - Interface Definition

```
iface eth0 inet dhcp
```

| অংশ | মানে |
|-----|------|
| `iface` | Interface define করছি |
| `eth0` | Interface-এর নাম |
| `inet` | IPv4 (inet6 হলে IPv6) |
| `dhcp` | IP automatically নেবে router থেকে |


#### Static IP Configuration

```
auto eth0
iface eth0 inet static
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 8.8.8.8 8.8.4.4
    dns-search devopsquad.com
```

| Line | মানে |
|------|------|
| `inet static` | Fixed IP (DHCP না) |
| `address` | এই machine-এর IP |
| `netmask` | Network mask |
| `gateway` | Router-এর IP |
| `dns-nameservers` | DNS servers |
| `dns-search` | Search domain |


#### Multiple Interface Configuration

```bash
# Loopback
auto lo
iface lo inet loopback

# First interface - DHCP
auto eth0
iface eth0 inet dhcp

# Second interface - Static IP
auto eth1
iface eth1 inet static
    address 10.0.0.5
    netmask 255.255.255.0
    gateway 10.0.0.1
```


### Interface Bring Up/Down করা

```bash
# Interface ডিজেবল করা
sudo ifup eth0

# Interface এনাবল করা
sudo ifdown eth0

# Restart করুন
sudo ifdown eth0 && sudo ifup eth0
```


## Modern Alternative: Netplan (Ubuntu 18.04+)

Modern Ubuntu server-এ `/etc/network/interfaces`-এর বদলে **Netplan** use হয়।

```bash
# Netplan config দেখুন
cat /etc/netplan/00-installer-config.yaml
```

**Output:**
```yaml
network:
  ethernets:
    eth0:
      dhcp4: true
  version: 2
```

**Static IP Netplan দিয়ে:**

```yaml
network:
  version: 2
  ethernets:
    eth0:
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
        search: [devopsquad.com]
      dhcp4: false
```

```bash
# Apply করুন
sudo netplan apply

# Test করুন (apply না করে শুধু check)
sudo netplan try
```


## তিনটি File একসাথে কীভাবে কাজ করে?

```
আপনি browser-এ লিখলে: google.com
           |
           ↓
   /etc/hosts check করো
   (google.com entry আছে? না।)
           |
           ↓
   /etc/resolv.conf দেখো
   (কোন DNS server use করবো? 8.8.8.8)
           |
           ↓
   Network interface use করে DNS query পাঠাও
   (/etc/network/interfaces বা Netplan দিয়ে configured eth0)
           |
           ↓
   DNS Server reply দিলো: 142.250.190.46
           |
           ↓
   Browser সেই IP-তে connect করলো
```


## Real DevOps Scenarios

### Scenario 1: CI/CD Server-এ Internal Services

```bash
sudo nano /etc/hosts
```
```
10.0.0.10    jenkins.internal
10.0.0.11    nexus.internal
10.0.0.12    gitlab.internal
```
এখন Jenkins pipeline-এ `nexus.internal` লিখলেই কাজ হবে।


### Scenario 2: DNS Troubleshooting

```bash
# DNS কাজ করছে কিনা check
nslookup google.com

# কোন DNS server use হচ্ছে?
cat /etc/resolv.conf

# DNS cache flush করুন
sudo systemd-resolve --flush-caches

# Resolution test করুন
resolvectl query google.com
```


### Scenario 3: Network Interface Debug

```bash
# Interface-এর current state দেখুন
ip addr show

# Routing table দেখুন
ip route show

# Default gateway কোনটা?
ip route | grep default
```


## 📝 Quick Summary

- **`/etc/hosts`** হলো local DNS table - DNS server-এর আগে check হয়, manual IP-to-name mapping করা যায়
- **`/etc/resolv.conf`** DNS resolver configure করে - কোন DNS server use হবে, search domain কী হবে
- **`/etc/network/interfaces`** network interface configure করে - DHCP বা Static IP, gateway, DNS
- Modern Ubuntu-তে **Netplan** এবং **systemd-resolved** এই কাজগুলো করে
- Resolution order: `/etc/hosts` → DNS server (যা `/etc/resolv.conf`-এ আছে)
