# 🐧 Linux For DevOps

> A structured, hands-on Linux learning journey built for DevOps Engineers who want to go beyond the basics and truly master Linux from the ground up.

## 📖 About This Repository

This repository is my personal Linux learning lab. It contains structured notes, command references, real examples, and hands-on exercises covering everything a DevOps Engineer needs to know about Linux from fundamentals to advanced kernel-level topics.

Every lesson is written in simple language with real-world DevOps context so that concepts are not just memorized but actually understood and applied.

## 🎯 Purpose

Most DevOps Engineers use Linux every day but rarely understand what's happening under the hood. This repo is built to change that. The goal is to:

- Build a **rock-solid Linux foundation** from scratch
- Understand how Linux works **at the system level**
- Learn tools and commands used in **real production environments**
- Be the engineer who knows **why** something works, not just **how**
- Prepare for certifications like **LFCS**, **LFCE**, and **CKA**

## 🛠️ How to Practice

Each lesson folder contains:
- **README.md** - Full lesson notes, concept explanations, and command references
- **examples/** - Real command examples with expected outputs *(where applicable)*
- **exercises/** - Hands-on tasks to practice on your own machine

### Recommended Practice Setup

1. **Spin up a Linux VM** using one of these options:
   - [Multipass](https://multipass.run/) - Lightweight Ubuntu VMs
   - [VirtualBox](https://www.virtualbox.org/) + Ubuntu ISO
   - [Vagrant](https://www.vagrantup.com/) - Reproducible VM environments
   - Any cloud VM (AWS EC2, GCP, DigitalOcean)

2. **Clone this repo** to your local machine or VM:
   ```bash
   git clone https://github.com/DevOpsquadHQ/Linux-For-DevOps.git
   cd Linux-For-DevOps
   ```

3. Open the lesson folder you want to study and read the README

4. Try every command yourself, reading alone is not enough

5. Complete the exercises at the end of each lesson before moving on

> 💡 **Pro Tip:** Keep a terminal and the lesson README open side by side. Type every command manually, don't copy-paste. Muscle memory matters.
